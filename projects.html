Projects

A few portfolio pieces that show how I plan pages before I build them.

Project Selection: The Projects page will focus on examples of work that show what I can build and how I think through a problem. Rather than listing every assignment without explanation, this page will feature a smaller number of projects with enough description to make each one meaningful. Visitors should be able to see what the project was, what tools I used, what challenge I solved, and what I learned from the process.

What the Work Shows

What the Work Shows: Each project summary will help answer practical questions that a visitor might have. Can I create clear layouts? Can I organize content so that it is easy to read? Can I follow a design plan and still make thoughtful choices? The Projects page is where those answers become visible. I want to include short descriptions of responsibilities, screenshots, and notes about revisions so the page communicates both final results and the thinking behind them.
